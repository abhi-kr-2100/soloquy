Things to Worry About Later

Operational concerns for the Soloquy backend deploy that we are deferring to keep the first cut simple. Not implemented yet — revisit before relying on this in production.

Critical (solve before launch)

  • Update path. user_data only pulls the image at boot, so a new release won't reach the running VM. Need a mechanism: recreate the instance on each deploy (brief downtime), a systemd timer that podman pulls on digest change, or a watchtower-style sidecar.
  • TLS / HTTPS. Currently plaintext :8080. The frontend needs HTTPS — terminate at a load balancer or a Caddy/Nginx reverse proxy with Let's Encrypt.
  • Secrets & config. App config (DB URL, API keys) needs a home: use OCI Vault fetched via Instance Principal. Don't bake into the image or Terraform.
  • Persistence. Always-Free instances can be reclaimed and local disk is ephemeral. Any state needs a block volume or Object Storage, or it is lost on reclaim/reboot.
  • Observability (no SSH). Can't podman logs in. Ship container logs to OCI Logging or the serial console before you need to debug a boot crash.

Should-have

  • Health checks. Restart=always covers process death, but add an OCI/LB HTTP health check on a real health endpoint so traffic stops if the app is wedged.
  • Rate limiting / abuse. VM is raw on :8080 with no WAF. Add edge rate limiting (LB / WAF or reverse proxy).
  • Zero-downtime updates. Single instance means any update is downtime. Acceptable for now; a LB + 2 instances later fixes it.
  • OCIR hygiene. Old image tags accumulate — set a retention/cleanup policy.
  • Cost guardrails. Free tier is easy to exceed (egress, provisioned LB, out-of-region pulls). Watch tenancy limits.

Don't bother yet

  • Auto scaling. Not possible on a single Always-Free A1 VM (no ASG, no reliable second free instance). Means moving off free tier + LB + multiple instances + shared DB. Ignore for now.
  • DNS. Static public IP is marked optional; add a stable domain later when needed.
Deferred items — not part of the initial simple deploy.
